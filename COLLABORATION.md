# コラボレーションガイドライン

## このリポジトリについて

このリポジトリは、共同開発（コラボレート開発）のための資料やベストプラクティスを提供することを目的としています。

## 編集ポリシー

### 誰が編集できるか

- ✅ **リポジトリオーナー（作成者）**: すべての編集が可能
- ❌ **一般ユーザー**: 直接の編集は不可

### なぜこの制限があるのか

1. **品質管理**: 内容の一貫性と品質を保つため
2. **正確性の維持**: 誤った情報の混入を防ぐため
3. **構成の統一**: ドキュメント構成を統一するため

## 貢献する方法

編集権限がない場合でも、以下の方法で貢献できます：

### 1. Issueの作成

誤りを発見した場合や改善提案がある場合：

1. Issuesタブを開く
2. 「New Issue」をクリック
3. 問題や提案を具体的に記述
4. 提出

### 2. Fork & Pull Request

コードや文書の変更を提案する場合：

1. リポジトリをフォーク（右上の「Fork」ボタン）
2. フォークしたリポジトリで変更を加える
3. 変更をコミット
4. オリジナルリポジトリにPull Requestを作成
5. オーナーによるレビューを待つ

### 3. ディスカッション

- GitHubのDiscussionsが有効な場合、そこでアイデアを共有
- または、Issueでディスカッションを開始

## 推奨される貢献内容

以下のような貢献を歓迎します：

### ドキュメント改善

- 誤字脱字の修正
- 説明の明確化
- 新しい例の追加
- 図や表の追加

### 新機能の提案

- 新しいトピックの追加
- 新しいガイドラインの提案
- ベストプラクティスの共有

### バグ報告

- リンク切れの報告
- 誤った情報の指摘
- 古くなった情報の指摘

## Issue作成時のテンプレート

### バグ報告

```
**問題の説明**
（何が問題か）

**場所**
ファイル名: 
行番号: 

**期待される内容**
（どうあるべきか）

**その他の情報**
（スクリーンショットなど）
```

### 機能提案

```
**提案内容**
（何を追加・変更したいか）

**理由**
（なぜ必要か）

**参考情報**
（関連リンクなど）
```

## レビュープロセス

Pull Requestが作成された場合：

1. **オーナーによるレビュー**
   - 内容の正確性確認
   - 既存ドキュメントとの整合性確認
   - コーディングスタイル確認（該当する場合）

2. **フィードバック**
   - 必要に応じて変更依頼
   - コメントでのディスカッション

3. **マージ**
   - 承認されたらマージ
   - 貢献者に感謝のメッセージ

## 行動規範

このプロジェクトに参加するすべての人は、以下を守ってください：

- 🤝 **尊重**: お互いを尊重し、建設的なコミュニケーション
- 📚 **学習**: 間違いは学習の機会として捉える
- 🎯 **焦点**: トピックに関連した議論を心がける
- 🌟 **ポジティブ**: 肯定的で支援的な態度

- タイトルに「質問:」と明記すると分かりやすいです
