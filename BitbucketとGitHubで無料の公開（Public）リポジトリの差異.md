# BitbucketとGitHubで無料の**Publicリポジトリ**を使用する際の違い
最も大きな違いはストレージ容量とチーム開発（コラボレーター）の制限です。

## **1. 無料公開リポジトリの制限比較**(2025年時点)
|特徴・制限|Bitbucket (Free)|GitHub (Free)|
|:---|:---|:---|
|全体ストレージ|合計 1GB (ワークスペース単位)|実質無制限 (1リポ100GBまで)|
|コラボレーター数|最大5人 (無料枠)|無制限|
|CI/CD (ビルド)|50分/月 (Pipelines)|2,000分/月 (Actions)|
|ファイルサイズ|100MBまで(LFS推奨)|100MBまで(LFS推奨)|
|その他制限|スニペット不可、DL不可|GitHub Pages機能あり|

## **2. 主な違いの詳細**
### 2.1. ストレージ容量の制限（Bitbucketが厳しい）
<dl>
<dt>Bitbucket:</dt> 
    <dd> 無料版のワークスペースは、すべてのリポジトリを合計して1GBしかストレージを使えません。これを超えるとリポジトリが読み取り専用になるなど大きな制限がかかります。</dd>
</dl>
<dl>
<dt>GitHub:</dt>
    <dd>公開リポジトリであれば、1リポジトリあたり100GBまで（全体での厳しい制限なし）と非常に寛大です。</dd> 
</dl>

### 2.2. コラボレーター数（Bitbucketは少人数向け）
<dl>
<dt>Bitbucket:</dt>
    <dd>無料プランはチームメンバーを5人までしか追加できません。</dd>
</dl>
<dl>
<dt>GitHub:</dt>
    <dd>無料プランでも無制限のコラボレーターを招待できるため、オープンソースプロジェクトに向いています。</dd> 
</dl>

### 2.3. CI/CD機能 (Pipelines vs Actions)
<dl>
<dt>Bitbucket:</dt>
    <dd>Pipelines（CI/CD）の無料枠は月間50分と非常に少ないです。</dd>
</dl>
<dl>
<dt>GitHub:</dt>
    <dd>Actionsの無料枠は月間2,000分あり、ビルドや自動テストを頻繁に行う場合に有利です。</dd> 
</dl>

### 2.4. その他（ダウンロード機能など）
<dl>
<dt>Bitbucket:</dt>
    <dd>無料のワークスペースでは、リポジトリのzipダウンロードや追加のファイルアップロードが制限されるようになりました。</dd>
</dl>
<dl>
<dt>GitHub:</dt>
    <dd>GitHub Pagesを利用してWebサイトを公開する機能が標準で使えます。</dd>
</dl> 

## 結論：どちらを選ぶべきか
<dl>
<dt>GitHubを選ぶべき:</dt>
    <dd>大規模なプロジェクト、オープンソース活動、頻繁なCI/CDが必要、大容量データを扱う。</dd>
</dl>
<dl>
<dt>Bitbucketを選ぶべき:<dt>
    <dd>5人以下の小規模チーム、Jiraソフトウェアと緊密に連携したい。</dd> 
</dl>
