# CollaborationSettings

コラボレート開発（共同開発）のための資料集

## 概要

このリポジトリは公開リポジトリですが、リポジトリ作成者のみが編集できる設定になっています。

## リポジトリの設定

- **公開設定**: Public（一般公開）
- **編集権限**: リポジトリ作成者のみ
- **閲覧権限**: 誰でも可能

## リポジトリの目的

共同開発のための資料やガイドラインを一般に公開しながら、内容の品質管理のため編集権限をリポジトリ作成者に限定しています。

## 設定方法

GitHubリポジトリをこの設定にする方法については、[SETTINGS.md](SETTINGS.md)を参照してください。

## Issueテンプレートの利用（他リポジトリへの導入）

このリポジトリには、GitHub Issue テンプレートが `.github/ISSUE_TEMPLATE/` 配下に格納されています。
他のリポジトリでも同じテンプレートを使いたい場合は、以下の手順でZIPから必要なフォルダだけを取り出して配置してください。

### 手順（ZIPを使ってテンプレートフォルダだけ取得する）

1. このリポジトリのトップページを開く
2. 右上付近の **`Code`** ボタンを押す
3. **`Download ZIP`** を選択してダウンロードする
4. ダウンロードしたZIPを展開する
5. 展開したフォルダ内から、次のフォルダを探す
   - `.github/ISSUE_TEMPLATE/`
6. 利用したい導入先リポジトリの直下に、以下のパスでコピーする
   - `<導入先リポジトリ>/.github/ISSUE_TEMPLATE/

   ※ `.github` フォルダが無い場合は新規作成してください。
   ※ 既に同名ファイルがある場合は、上書きするか、名前を変更して併存させてください。

7. 導入先リポジトリにコミット＆pushする（例：Gitを使う場合）

   ```bash
   git add .github/ISSUE_TEMPLATE
   git commit -m "Add issue templates"
   git push
   ```

8. GitHub上で導入先リポジトリの `Issues` → `New issue` を開き、テンプレートが選択肢に表示されることを確認する

### 補足・注意点

- テンプレートは導入先リポジトリの **デフォルトブランチ（通常 `main`）** に入っている必要があります。
- テンプレートが表示されない場合は、以下を確認してください。
  - 配置先が **`.github/ISSUE_TEMPLATE/`** になっているか（フォルダ名の大文字小文字も含めて一致）
  - push先がデフォルトブランチか
  - ブラウザの再読み込み（強制リロード）を実施したか
